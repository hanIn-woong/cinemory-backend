# 📄 요구사항 명세서 v2.0 (Requirements Specification)

## 1. 프로젝트 개요
### 1.1 프로젝트명
CineMory (시네모리) - 개인 맞춤형 영화 기록 및 큐레이션 웹 애플리케이션

### 1.2 기획 목적
미디어 소비량이 급증하는 환경에서, 개개인의 영화 감상을 단순 기록하는 것을 넘어 데이터를 다각도로 시각화하고 취향을 분석하여 제공하는 고도화된 맞춤형 아카이빙 시스템을 구축한다.

### 1.3 타겟 사용자
* 다회차 관람을 즐기며 체계적인 감상 기록을 남기고 싶은 영화 애호가
* 자신의 영화 시청 통계와 취향 데이터를 직관적으로 확인하고 싶은 사용자
* 깔끔하고 모던한 UI를 통한 데이터 정리를 추구하는 사용자

---

## 2. 개발 환경 및 기술 스택
* **Frontend:** React, JavaScript (axios/fetch), CSS Modules / Styled-components
* **Backend:** Java (JDK 21), Spring Boot, Spring Data JPA, Gradle
* **Database:** MySQL
* **External API:** TMDB (The Movie Database) Open API
* **IDE:** IntelliJ IDEA

---

## 3. 기능적 요구사항 (Functional Requirements)

### [FR-01] 실시간 영화 검색
* **FR-01-01:** 사용자는 검색창에 영화 제목을 입력하여 영화를 검색할 수 있다.
* **FR-01-02:** 시스템은 TMDB API를 호출하여 영화 메타데이터(포스터, 장르, 러닝타임 등)를 실시간으로 가져온다.
* **FR-01-03:** 검색 결과는 React 컴포넌트를 통해 페이지 새로고침 없이 비동기적으로 렌더링된다.

### [FR-02] 다회차 시청 기록 아카이빙 (1:N 데이터 관리)
* **FR-02-01:** 사용자는 시청한 영화에 대해 별점과 감상평을 작성하고 자체 DB에 저장할 수 있다.
* **FR-02-02:** 동일한 영화에 대해 여러 번의 시청 기록(N회차 관람)을 남길 수 있으며, 각 기록마다 독립적인 시청 날짜와 리뷰를 부여할 수 있다.

### [FR-03] 맞춤형 컬렉션 생성 및 관리 (N:M 데이터 관리)
* **FR-03-01:** 사용자는 자신의 기준에 따라 새로운 '컬렉션(폴더)'을 생성할 수 있다.
* **FR-03-02:** 시청하거나 검색한 영화를 특정 컬렉션에 추가/제외할 수 있으며, 하나의 영화가 여러 컬렉션에 포함되는 다대다(N:M) 관계를 지원한다.

### [FR-04] 사용자 데이터 분석 및 시각화 (My Cine-Stats)
* **FR-04-01:** DB에 축적된 사용자의 시청 기록을 기반으로 선호 장르, 선호 감독 등의 취향 통계를 차트(방사형, 파이 차트 등)로 시각화하여 제공한다.
* **FR-04-02:** 시청한 영화들의 러닝타임 데이터를 합산하여 총 누적 시청 시간을 계산 및 표시한다.

### [FR-05] 시네마 캘린더 및 연말 결산
* **FR-05-01:** 달력 형태의 UI를 제공하며, 시청 기록이 있는 날짜에 해당 영화의 썸네일(포스터)을 도장처럼 표시한다.
* **FR-05-02:** 사용자의 1년 치 통계와 캘린더 데이터를 종합하여 '연말 결산' 이미지를 생성하고 공유할 수 있는 기능을 제공한다.

### [FR-06] 취향 기반 영화 추천 (Cold Start 방지)
* **FR-06-01:** 사용자가 별점 4점 이상을 부여한 영화 목록을 바탕으로, TMDB 유사 영화 API를 활용하여 맞춤형 영화를 추천한다.
* **FR-06-02:** 타 사용자의 데이터 없이도(Cold Start 문제 해결) 개인의 기록만으로 정교한 콘텐츠 기반 필터링(Content-based Filtering) 추천을 수행한다.

## 4. 비기능적 요구사항 (Non-Functional Requirements)

### [NFR-01] 성능 및 사용성
* **NFR-01-01:** 프론트엔드(React)와 백엔드(Spring Boot) 간의 통신은 RESTful API를 기반으로 처리하여 빠른 응답성을 보장한다.
* **NFR-01-02:** React의 SPA(Single Page Application) 특성을 살려 매끄러운 화면 전환과 사용자 경험(UX)을 제공한다.

### [NFR-02] 시스템 아키텍처 및 데이터 무결성
* **NFR-02-01:** 백엔드 서버와 프론트엔드 서버를 완전히 분리하여 모듈 간 결합도를 낮추고 유지보수성을 극대화한다. (CORS 설정 필수 적용)
* **NFR-02-02:** 외부 API(TMDB)에서 제공하는 데이터와 내부 시스템에서 생성되는 사용자 데이터(시청 기록, 캘린더, 통계)를 분리하여 데이터 무결성을 확보한다.
